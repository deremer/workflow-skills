What it is

A meta-skill that builds other skills. It runs a four-phase, human-gated pipeline that interviews you about your process, designs the architecture, generates every file, and validates the result against the open standard. The work stops at a gate between each phase and waits for your explicit approval before moving on, so you stay in control as the design takes shape.

01
Scope
Interview for intent, phases, platforms, constraints.
02
Architect
Design state contracts, gates, execution models.
03
Build
Generate the orchestrator and all phase files.
04
Validate
Run the checklist, test, package for delivery.
Gate between every phase. Never skipped, never optional. The skill presents what it built, names what you should judge, and waits.

When it earns its keep

Use a workflow only when all four hold at once:

3+ distinct phases producing different artifacts
· a human needs to review between phases
· each phase depends on the one before
· the process repeats often enough to encode

If a single prompt would do, the skill tells you to use that instead.

Key takeaways

  • Front-load planning. The cheapest place to fix a mistake is before any work starts.
  • Gate autonomy with review. A human checkpoint wherever being wrong is costly to undo.
  • Make handoffs contracts. Name the artifact, its format, and its structure — not "the output from before."
  • Bound every loop. A set revision and research limit beats "refine until it's good."
  • Verify before done. A self-check that runs until green beats a confident claim.
  • Capture the why. State blocks carry decisions so a resumed run inherits them.
